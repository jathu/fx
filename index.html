Quickly create developer friendly CLI tools.


fx is a workspace tool manager. It allows you to create consistent, discoverable, language-neutral and developer friendly command line tools.

fx is a command line tool (CLI) that hosts and manages other CLIs. It takes care of the tedious parts of CLIs like argument parsing, validation, help/man pages and updates. fx has dynamic subcommands based on its current directory — these subcommands are created by you, the user. Still don't understand? Want to learn more? Walk through the step-by-step tutorial below.

Example

Suppose our large repository has multiple languages. Let's create a code formatter to support everything via fx tools/format.

Let's create a command named tools/format and describe its interface:

~/app $ 
vim tools/format/command.fx.yaml

Now, let's create the respective script in Python. The parsed arguments will be passed into the script as a single JSON argument.

The command now exists for everyone and is easily discoverable.

~/app $ 
fx

It automatically includes a developer friendly help page.

~/app $ 
fx tools/format --help

fx tools/format — Format and analyze code.


usage: fx tools/format [-t|--test] [-l|--language=<language...>]


OPTIONS

--test, -t · bool · optional

Format test without overwriting.

Default: false


--language, -l · List<string> · optional

Format specific languages.

Default: all | Choices: all, cpp, c++, bazel
